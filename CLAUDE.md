# Правила репозитория «Учёт финансов»

## Скрипты .bat / .cmd — только ASCII

В файлах `*.bat` и `*.cmd` запрещены любые не-ASCII символы (включая кириллицу, ё, №,
кавычки-«ёлочки», длинные тире). Все сообщения `echo` — только латиницей/транслитом.

Причина: `cmd.exe` исполняет .bat в OEM-кодовой странице (cp866 в русской Windows),
а редакторы сохраняют файлы в UTF-8/cp1251. Кириллица превращается в кракозябры,
а в командах/путях/`if`-условиях может ломать исполнение скрипта.

Проверка автоматизирована в CI (`.github/workflows/ci.yml`, шаг «Check bat files are ASCII-only»).

## Прочее

- Перед изменением схемы БД — миграция с сохранением данных (паттерн в `src/main/database.ts`, блок «Schema migrations»).
- Не трогать без явного запроса: формулу кассового потока `J(n) = K(n-1) + dailyBudget`,
  replay-механизм платежей «Папа» (`replayDadPayments`), алгоритм `debtAlgorithm.ts`.
- Удаление накопительного счёта намеренно сохраняет связанные операции в журнале — это
  подтверждённое пользователем поведение, не «чинить».
- На каждый исправленный финансовый баг — юнит-тест (vitest, чистые функции в `src/main/finance.ts`).
- Каждое изменение, которое собирается в релиз, обязано поднимать `version` в `package.json` (semver);
  релиз публикуется тегом `vX.Y.Z`, совпадающим с этой версией.
- Артефакты сборки (`out/`, `dist/`, `*.tsbuildinfo`) не коммитятся.
